Google Dorking : Trouver les pages Admin

Requête utilisée : inurl:admin site:demo-vulnerable.com

Qu'est-ce que le Google Dorking ?

Le Google Dorking (aussi appelé Google Hacking) est une méthode de recherche avancée qui utilise des opérateurs spécifiques dans Google pour trouver des informations sensibles ou cachées publiquement sur le web. Cette technique est très utilisée dans la phase de reconnaissance passive lors d’un test d’intrusion.

Décomposition de la commande

inurl:admin site:demo-vulnerable.com
  • inurl:admin : recherche dans les URLs contenant le mot "admin" (ex: /admin/, /admin-login.php, etc.).
  • site:demo-vulnerable.com : limite la recherche au site spécifique ciblé.

Exemples de résultats trouvés

  • http://demo-vulnerable.com/admin/
  • http://demo-vulnerable.com/admin-login.php
  • http://demo-vulnerable.com/site/admin_panel/

Ces pages peuvent pointer vers des interfaces de connexion d'administration potentiellement vulnérables (injection SQL, accès non protégé, etc.).

Pourquoi utiliser cette méthode ?

  • Détecter rapidement des interfaces d'administration accessibles publiquement.
  • Rester en mode passif (aucun scan direct sur la cible).
  • Collecter des URL intéressantes à tester avec des outils comme Burp Suite ou SQLMap.

Autres Dorks utiles

  • intitle:"index of" site:demo-vulnerable.com : recherche de répertoires ouverts.
  • filetype:sql site:demo-vulnerable.com : fichiers de bases de données exposés.
  • intext:"password" site:demo-vulnerable.com : mots de passe visibles dans des pages ou fichiers.

Pourquoi c'est un risque pour l'entreprise ?

Beaucoup de sites web indexent par erreur des URLs sensibles ou fichiers de configuration. Les moteurs de recherche les rendent accessibles publiquement si les règles de sécurité ne sont pas correctement configurées.

  • Oubli de fichiers de backup sur le serveur.
  • Mauvais paramétrage du fichier robots.txt.
  • Pas d’authentification sur des interfaces critiques.

Comment se protéger ?

  • Configurer le fichier robots.txt pour interdire l’indexation de certains répertoires.
  • Appliquer des règles d’accès avec authentification forte sur les pages sensibles.
  • Utiliser un WAF (Web Application Firewall) pour filtrer les accès.
  • Surveiller régulièrement ce qui est indexé par Google avec : site:example.com
