Utilisation de theHarvester

Commande : theHarvester -d demo-vulnerable.com -b google -l 100

Qu'est-ce que theHarvester ?

theHarvester est un outil de reconnaissance passive très utilisé en phase initiale de test d'intrusion. Il permet de récolter des informations publiques sur un domaine cible comme des adresses email, des noms d'utilisateur, des sous-domaines ou encore des adresses IP.

Commande analysée

theHarvester -d demo-vulnerable.com -b google -l 100

Détail des options utilisées :

  • -d : spécifie le domaine cible à analyser (demo-vulnerable.com).
  • -b : définit le moteur de recherche utilisé pour collecter les données (ici google).
  • -l : limite le nombre de résultats retournés (ici, 100).

Exemple de Résultat

Après exécution, on peut obtenir des informations comme :

[-] Emails found:
  - contact@demo-vulnerable.com
  - admin@demo-vulnerable.com
  - hr@demo-vulnerable.com

[-] Hosts found:
  - www.demo-vulnerable.com
  - dev.demo-vulnerable.com
  - mail.demo-vulnerable.com

Ces résultats sont précieux pour cibler de futurs vecteurs d'attaque comme :

  • Campagnes de phishing ciblées avec les emails récoltés.
  • Recherche de services ou applications hébergées sur les sous-domaines trouvés.
  • Reconnaissance approfondie sur les serveurs de messagerie.

Pourquoi utiliser theHarvester ?

  • Collecte rapide d'informations publiques pour éviter d'éveiller les soupçons (mode passif).
  • Utile pour préparer une phase d'ingénierie sociale ou pour repérer des services en ligne oubliés.
  • Supporte de nombreuses sources de données : Google, Bing, Yahoo, Baidu, LinkedIn, GitHub, etc.

Bonnes pratiques et conseils

  • Tester plusieurs moteurs de recherche pour diversifier les résultats : -b bing, -b linkedin, etc.
  • Exporter les résultats avec -f html pour les lire plus facilement :
theHarvester -d demo-vulnerable.com -b google -l 100 -f resultat.html
  • Comparer les résultats avec ceux d'autres outils comme Amass ou Recon-ng.

Conclusion

theHarvester est un excellent outil pour enrichir la phase de collecte d’informations lors d’un test d’intrusion. Facile à utiliser, il permet de découvrir rapidement des éléments critiques sur la cible à partir de sources publiques. Une bonne utilisation de cet outil permet d’anticiper les vecteurs d’attaque avec précision.
