Nmap (Network Mapper) est un outil open-source pour analyser les réseaux, découvrir les hôtes actifs, scanner les ports et identifier les services.

Attention : N'utilisez Nmap que sur des réseaux autorisés.

Commandes de base

Commande Description
nmap -sn 192.168.1.0/24 Scan des hôtes actifs. L'option -sn (pour scan no port) dit à Nmap de ne PAS scanner les ports, mais simplement de voir quels hôtes sont en ligne sur un réseau. C'est une méthode rapide pour détecter les machines actives, sans vérifier quels services tournent dessus.
nmap -sS 192.168.1.1 Scan furtif (SYN). Technique de scan de ports utilisée pour éviter d'être détecté par un pare-feu ou un système de détection d'intrusion (IDS).
nmap -sV 192.168.1.1 Détection des versions des services. Permet de détecter la version exacte des services qui tournent sur les ports ouverts d'une machine. Tu ne vois pas seulement que le port 80 est ouvert... tu sais si c'est Apache 2.4.25 ou nginx 1.18, par exemple.
nmap -O 192.168.1.1 Détection de l'OS. Permet à Nmap d'estimer le système d'exploitation de la machine distante (Windows, Linux, BSD, etc.), ainsi que des détails comme la version du noyau ou le type d'appareil (routeur, imprimante...). Il analyse la manière dont la cible répond à certains paquets TCP/IP pour deviner l'OS.
nmap -A 192.168.1.1 Scan agressif (OS + versions). Combine plusieurs fonctions puissantes de Nmap pour fournir une analyse complète d'une cible. C'est comme une commande « tout-en-un » pour récolter un maximum d'informations.
nmap -p- 192.168.1.1 Scan tous les ports (1-65535). Par défaut, Nmap ne scanne que les 1 000 ports les plus courants. Mais avec -p-, tu lui dis :
"Scanne absolument tous les ports TCP, de 1 à 65535 !"

Bonnes pratiques

  • Utilisez -sS pour des scans discrets.
  • Évitez les scans agressifs (-T4/T5) sur des réseaux sensibles.
  • Sauvegardez les résultats avec -oN scan.txt ou -oX scan.xml.
  • Testez toujours en environnement contrôlé (machines virtuelles, labs).

Gobuster est un outil puissant utilisé pour découvrir rapidement des dossiers, fichiers et sous-domaines cachés sur un site web ou un domaine.

Énumération web

  • Découvrir des dossiers et fichiers cachés
  • Trouver des pages d'administration
  • Identifier des fichiers sensibles oubliés

Énumération DNS

  • Détecter des sous-domaines
  • Découvrir des services cachés
  • Cartographier l'infrastructure

Pourquoi c'est important ?

En test de sécurité, Gobuster permet de trouver des points d'entrée cachés :

  • Interfaces d'administration non liées depuis le site principal
  • Fichiers de sauvegarde (.bak, .old) contenant des données sensibles
  • Sous-domaines de développement (dev., staging.) potentiellement vulnérables
  • API internes exposées par erreur

Avantage clé : Gobuster automatise cette recherche fastidieuse qui serait impossible à faire manuellement, tout en étant plus discret qu'un scanner graphique.

Attention : N'utilisez Gobuster que sur des systèmes pour lesquels vous avez une autorisation écrite.

Commandes essentielles

Commande Description
gobuster dir -u http://site.com -w wordlist.txt Scan de répertoires web
-u : URL cible
-w : Wordlist à utiliser
gobuster dns -d example.com -w subdomains.txt Découverte de sous-domaines
-d : Domaine cible
Résolution DNS avec serveurs par défaut
gobuster vhost -u http://site.com -w hosts.txt Détection d'hôtes virtuels
Utile pour trouver des sous-domaines non résolus par DNS

Options importantes

Option Description
-x php,html,txt Extensions à tester (séparées par des virgules)
-t 50 Nombre de threads (par défaut: 10)
-o result.txt Sauvegarde des résultats dans un fichier
-k Ignore la vérification du certificat SSL
-s 200,204 Codes HTTP à considérer comme valides

Bonnes pratiques

  • Utilisez des wordlists adaptées (common.txt, directory-list-2.3-medium.txt)
  • Limitez le débit avec --delay pour éviter le bannissement
  • Testez différentes extensions (-x php,aspx,jsp)
  • Utilisez -k pour les sites avec SSL/TLS
  • Documentez toujours vos résultats avec -o

Définition

Dirb (Directory Brute-forcer) est un scanner web qui recherche :

  • Répertoires cachés sur un serveur web
  • Fichiers sensibles (config, backups)
  • Pages d'administration non liées

Différence avec Gobuster

  • Plus ancien mais toujours efficace
  • Inclus par défaut dans Kali Linux
  • Wordlists intégrées
  • Moins rapide que Gobuster

Cas d'usage typiques

  • Trouver /admin ou /backup
  • Détecter des fichiers .git exposés
  • Identifier des pages de login cachées
  • Trouver des fichiers robots.txt ou phpinfo.php

Attention : Dirb peut générer beaucoup de requêtes - utilisez -z pour limiter le débit.

Commandes essentielles

Commande Description
dirb http://site.com Scan de base avec la wordlist par défaut
dirb http://site.com /usr/share/wordlists/dirb/common.txt Avec une wordlist spécifique
dirb http://site.com -X .php,.bak Recherche avec extensions spécifiques
dirb http://site.com -N 302 Ignore les redirections (code 302)
dirb http://site.com -u admin:password Authentification basique

Options Clés

🎯 Filtrage

  • -x .php,.html → Recherche uniquement les fichiers avec ces extensions.
    Ex : /login.php, /index.html
  • --hc 404 → Ignore les pages 404 (non trouvées).
    Ex : Ne montre pas les erreurs, seulement les vrais fichiers
  • --mc 200 → Affiche uniquement les réponses HTTP 200 (OK).
    Ex : Pages valides uniquement

⚡ Performance

  • -t 50 → Lance 50 threads pour un scan plus rapide.
    ⚠️ Plus de threads = plus rapide, mais plus bruyant !
  • -z 100ms → Attente de 100 millisecondes entre les requêtes.
    Permet de ne pas surcharger le serveur
  • -r → Désactive la récursivité.
    Utile si tu ne veux pas scanner les sous-dossiers trouvés

🔐 Sécurité & Authentification

  • -u admin:admin → Authentification HTTP basique (login:password).
    Ex : pour accéder à des zones protégées
  • -H "User-Agent: Mozilla" → Envoie un en-tête HTTP personnalisé.
    Ex : simuler un navigateur
  • -p http://127.0.0.1:8080 → Utilise un proxy (ex : Burp Suite)
    🔍 Pour analyser les requêtes dans un proxy

📦 Sortie & Affichage

  • -o result.txt → Sauvegarde les résultats dans un fichier.
    Ex : Pour les relire plus tard
  • -v → Affiche les détails en temps réel (mode verbeux).
    Utile pour voir la progression du scan
  • -s → Mode silencieux : affiche seulement les résultats positifs.
    Pour un affichage plus propre

Proxy Intercept

Sert à intercepter et modifier les requêtes HTTP/HTTPS en temps réel entre votre navigateur et le serveur. C'est le point d'entrée de la plupart des tests.

  • Analyse des headers, cookies, et paramètres GET/POST
  • Inspection fine de l’application (auth, tokens, injection...)
  • Essentiel pour du pentest en boîte noire
Contrôle total des échanges

Scanner de Vulnérabilités

Analyse automatiquement les failles les plus courantes sur vos cibles (XSS, SQLi, CSRF...).

  • Scan actif et passif
  • Rapports détaillés avec CVSS
  • Exploits reproductibles
Disponible uniquement en version Pro

Intruder

Permet de lancer des attaques automatisées en injectant des charges utiles (payloads) dans des points de test.

  • Test de force brute sur login ou tokens
  • Fuzzing d’entrées utilisateur
  • 4 modes d’attaque : Sniper, Battering ram, Pitchfork, Cluster bomb
Automatisation ciblée des attaques

Repeater

Outil pour rejouer manuellement des requêtes HTTP avec des modifications, utile pour tester des hypothèses ou des injections.

  • Analyse fine des réponses
  • Comparaison de résultats avec variations
  • Support des cookies et sessions
Fuzzing manuel avancé

Decoder

Utilisé pour convertir des données encodées dans divers formats (Base64, URL, HTML, Hex...) à des fins de test ou de compréhension.

  • Décode et encode les payloads
  • Transformation rapide de contenu suspect
  • Utile pour l’analyse de JWT, cookies, etc.
Pratique pour la rétro-ingénierie
